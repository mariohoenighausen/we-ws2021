1.1. Fachliche Argumentation über Erfolgsprinzipien des WWW

Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?

Es wurde als zu "vage" bzw. zu ungenau in Bezug auf die Evolvierbarkeit des Webs und den Umgang mit insbesondere "Broken Links" und referentieller Integrität gesehen und entsprach nicht dem damaligen akademischen Konsens bzw. wurde als degressive Entwicklung verkannt.

Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

Durch die Orthogonalität der Entwicklungsmöglichkeiten des Webs (zu Beginnn Interaktion == http , Representation == HTML und Identification == URI), sowie der nicht nötigen mathematischen Rigorosität am Beispiel der Möglichkeit von broken Links, die also kein größeres Verständnis von referentieller Integrität zum rudimentären Erstellen einer Webseite nötig machten.

Was wäre der Preis für die garantierte Verhinderung von “broken links”?

Die eingeschränkte, nicht orthogonale, zentralisierte Entwicklung und das Verhalten des Webs und seiner Technologien wären der Preis gewesen. Es müsste ebenso referentielle Integrietät garantiert werden.

1.2 HTTP

Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

Bei erfolgreichen http-Anfragen, wird der Code 200 zurückgegeben, welcher ein „OK“ symbolisiert. Die genaue Bedeutung hängt aber vom jeweiligen http-Verb ab.

Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

Die „Moved Permanently“ Antwort, beschreibt bei GET oder HEAD Anfragen, dass eine Ressource (HTML oder andere) auf eine andere URL verschoben wurde.

Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

Die „Bad Request“ Antwort beschreibt, dass der Server nicht antworten (die Ressource bzw. der Pfad angegebene Pfad zur Ressource existiert gar nicht) oder das durch Clientseitige Fehler die Anfrage nicht bearbeiten werden kann. Die Anfrage sollte, vom Client so verändert werden, dass sie korrekt formuliert ist. Browser Cache leeren

Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

De „Forbidden“ Antwort beschreibt, dass eine Anfrage vom Server erhalten und verstanden, aber nicht autorisiert wurde, also man nicht die nötigen Rechte besitzt, um auf die angefragte Ressource zugreifen zu dürfen. Durch das Wechseln der Authentifikationsrolle (kein wiederholtes Authentifizieren mit selben Privilegien, sondern mit anderen) oder das Erhalten von weiteren bzw. das Ändern der benötigten Rechte kann der Zugriff ermöglicht werden.

In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

Durch das Setzen von „Cache-Control“ im Header einer http-Request/Antwort.

1.3 HTML-Literatur lesen und Fragen

Was ist HTML?

HTML ist die Abkürzung für Hypertext Markup Language. Es beschreibt eine unstrukturierte Skriptsprache, die auf tags (Markern) basiert und zur Gestaltung des Aufbaus einer Webseite genutzt wird.

Wie kann man eine geschachtelte geordnete Liste der Schachtelungstiefe 3 erzeugen?

                                
<!DOCTYPE html>
<html>
    <ol>
        <li>1</li>
        <ol>
            <li>2</li>
            <ol><li>3</li></ol>
        </ol>
    </ol>
</html>

Wie ist eine HTML-Tabelle aufgebaut?

Eine HTML-Tabelle besteht aus den Tags <table> (definiert die Nutzung eines Tables), (definiert die Tabellen Zeile), <th> (definiert Kopfspalten) und <td> (definiert table data bzw. Spalten und Spaltenwerte).

Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?

Wo es möglich ist, sollten relative Dateipfade verwendet werden, damit man sichergeht, dass das Programm nicht nur auf der lokalen Entwicklungsmaschine durch das Benutzen von absoluten Pfaden, funktioniert. Des Weiteren sollten Dateinamen tendenziell klein geschriebene werden, da man dadurch Probleme mit Groß- und Kleinschreibungssensitiven Systemen vermeidet.

Wie baut man in HTML ein Menü?

Menüs werden durch das Verwenden des Navigations-Tags (<nav>), von ungeordneten Listen (<ul>), sowie Anchor Tags (<a href=““></a>) realisiert. Href kann auf eine id, oder http Adressen referenzieren. 
                                
<!DOCTYPE html>
<html>
<body>
    <nav>
        <ul>
            <li><a href=""></a></li>
            <li><a href=""></a></li>
            <li><a href=""></a></li>
        </ul>
    </nav>
</body>
</html>

Welche Attribute sollte man bei Bildern wie verwenden?

Bei Bildern (<img>) sollte man die Attribute src, zum Setzen einer Quelle, alt, zum Setzen eines bei nicht Darstellbarkeit des Bildes alternativ darzustellenden Textes, sowie width zum Setzen der Breite und height zum Sezten der Höhe eines Bildes, mindestens verwenden. <img src="" alt="Alternativer Text" height="" width="">
